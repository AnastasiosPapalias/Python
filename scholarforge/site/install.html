Install

One minute, any platform.

Python 3.8 or newer. Git. That is the whole prerequisite list.

Standard install

git clone https://github.com/AnastasiosPapalias/Python.git
cd Python/scholarforge
pip install -e .

# optional: PDF + DOCX extraction
pip install pypdf python-docx

# verify
scholarforge --version

Windows, first-time user

A walkthrough written for beta testers with no Python background is shipped as BETA-TESTER-GUIDE.md. Six steps, plain English, no jargon.

First project

mkdir my-research
scholarforge new my-research
# (wizard guides you through seven plain-English questions)

scholarforge search --project my-research "your title or question"
scholarforge export --project my-research --format markdown

API keys (optional)

ScholarForge works with zero keys. Two sources benefit from free keys for higher rate limits:

  • CORE — repository full-text aggregator.
  • Semantic Scholar — AI-ranked citation metadata.

Run scholarforge config --init to write a commented ~/.scholarforge.toml template, then fill in your keys.

Upgrading

cd Python/scholarforge
git pull
pip install -e .

Uninstall

pip uninstall scholarforge

Your project folders are plain files — delete them by hand if you want to.
